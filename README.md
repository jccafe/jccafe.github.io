# github_html - 網站目錄自動化生成與部署助手

這是一個為 `jccafe.github.io` 靜態網站開發的自動化管理工具，旨在將「網頁檔案掃描、二階層導覽目錄生成、本地與雲端 Git 提交推送工作流」全面自動化，大幅簡化靜態網站的維護成本。

---

## 🌟 核心特色

1. **現代化兩欄式導覽 UI**：
   * **桌機端**：左側為固定的磨砂玻璃資料夾導覽側邊欄 (Sidebar)；右側為分類網頁卡片區。
   * **行動端 (RWD)**：小螢幕下左側側邊欄自動轉換為**頂部水平滑動的導覽 Tab 欄**，提供極佳的行動端觸控體驗。
   * **平滑滾動與 Scrollspy (滾動選單追蹤)**：點擊左側目錄會平滑滾動至對應區塊；向下滾動時，左側選單會自動跟隨高亮，且在行動端會自動將選中的 Tab 橫向滾動對齊到螢幕中心。
   * **智慧搜尋過濾**：內建 JavaScript 即時搜尋。當搜尋某個關鍵字時，若某資料夾下無任何符合卡片，該整個資料夾區塊與左欄對應選項會自動隱藏，維持搜尋清爽。

2. **自動化部署腳本 (`deploy.py`)**：
   * 實現一鍵化：自動偵測當前分支，執行 `git pull` -> `generate_index.py` -> `git add/commit` -> `git push` 工作流。
   * 若無檔案變更，會自動略過 commit 與 push 優雅結束。

3. **衝突與安全防禦機制**：
   * 腳本內置 Windows 終端機 CP950 編碼自適應調整，保證中文字元列印不崩潰。
   * 自動執行 Git 預檢（如是否為 Git 儲存庫），且在 `git pull` 失敗或偵測到 Git 合併衝突時，立即中斷執行並以紅色字體發出警告，防止強制覆蓋代碼。

4. **GitHub Actions 雲端自動更新**：
   * 內建 CI/CD 自動化工作流。每當有新 HTML 網頁推送至遠端 `main` 分支時，GitHub 伺服器會自動在背景掃描網頁並更新 `index.html`，實現完全免手動的雲端同步。

## 🚀 使用指南

### 1. 環境準備
本專案使用 `uv` 進行虛擬環境管理。請確保本機已安裝 `uv` 與 `git`，並在專案根目錄下初始化虛擬環境：
```bash
# 初始化 Python 虛擬環境
uv venv
```

### 2. 手動重新生成目錄
如果您只想在本機端掃描並重新生成 `index.html`，請執行：
```bash
uv run generate_index.py
```

### 3. 一鍵本地更新與部署
如果您在本機新增了網頁檔案，想要將變更一次性同步並推送到遠端儲存庫，請執行：
```bash
uv run deploy.py
```
*這將會自動執行：拉取遠端最新變更 -> 生成目錄 -> 暫存變更 -> 提交 -> 推送。*

### 4. 遠端雲端自動更新
由於專案已配置 **GitHub Actions**：
* 您可以直接在 GitHub 網頁版上建立/編輯 HTML 檔案，或是使用 Git 將新網頁直接推送 (Push) 到 `main` 分支。
* GitHub 會自動在背景完成目錄更新並發布，您完全不需執行任何本機命令。
