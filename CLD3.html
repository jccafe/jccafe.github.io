CLD 練習 3：動作引擎計時器

Action Engine Timer - 功能性全域變數 (FGV) 練習

練習目標

使用 LabVIEW 中的 Get Date/Time in Seconds (取得日期/時間（秒）) 或 Tick Count (ms) (滴答計數) 計時 VI，並根據指定的人機介面開發一個 功能性全域變數 (FGV) 計時器。

一般運作流程

  • ● 從零開始累加至「時間目標」。
  • ● 持續更新「已過時間」指示器。
  • ● 時間到期時，「時間是否已過」LED 必須亮起。
  • ● 必須具備暫停 (Pause) 與恢復 (Resume) 功能。

人機介面術語

控制元件 說明
Timer Mode 用於 FGV 的列舉 (Enum) 狀態。
Time Target 計時目標秒數。
Elapsed Time 顯示當前已過秒數與毫秒。
Has Elapsed 到期指示 LED。

計時器模式 (Timer Modes)

Start Timer

使用設定的「時間目標」重新啟動計時程序。

Read Time

計算並傳回當前的已過時間與到期狀態。

Pause

暫停計時，保持當前時間數值，停止增加。

Resume

從先前暫停的計時點恢復運作。

思考問題 (Questions)

1. 有什麼方法可以解決位元計時器的「翻轉」(Turnover) 事件？

2. 「已過時間」可以用來實現運行總時間的累加嗎？

3. 使用時間戳記 (Time Stamp) 時，日期和年份重要嗎？

挑戰練習 (Challenge Exercise)

開發一個不具備暫停功能的計時器。

  • 僅包含兩個狀態：Elapsed (計時中) 與 Reset (重設)。
  • 時間目標 (Wait) 直接在啟動階段設定，而非作為獨立狀態。
