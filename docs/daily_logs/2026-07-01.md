## 錯誤與學習

1. **Windows 控制台 CP950 解碼錯誤 (UnicodeEncodeError)**：
   * **問題**：在 Python 腳本中使用 Unicode 表情符號（如 `✅`、`🚀`、`🎉` 等）在預設 CP950 編碼的 Windows 終端機印出時崩潰，報出 `UnicodeEncodeError: 'cp950' codec can't encode character...`。
   * **解決**：
     * 將所有 print 輸出中的表情符號替換為安全的 bracket 文字標記（如 `[OK]`, `[Deploy]`, `[Success]`）。
     * 在 Python 腳本開頭加入 stdout 重組編碼設定：`if hasattr(sys.stdout, 'reconfigure'): sys.stdout.reconfigure(encoding='utf-8')`。這確保了中文字元與其他 Unicode 字符能在 Windows 終端機順利列印。
     
2. **非 Git 儲存庫環境出錯**：
   * **問題**：若使用者在尚未初始化為 Git 儲存庫的資料夾中直接運行 `deploy.py`，Git 命令會崩潰並輸出大量英文 stack trace。
   * **解決**：在 `deploy.py` 中加入預檢步驟，檢查 `git rev-parse --is-inside-work-tree`。若非 Git 儲存庫，會輸出清晰、友善的繁體中文錯誤提示，指引使用者初始化與連線遠端，提升了系統強健性。
