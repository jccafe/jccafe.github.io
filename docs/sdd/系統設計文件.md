# github_html 系統設計文件 (SDD)

## 1. 技術選型 (Technology Stack)

* **運行環境**：Python 3.10+
* **環境管理**：`uv` (極速 Python 套件與虛擬環境管理器，依據全域規則)
* **版本控制與部署**：Git CLI (需要預先安裝於執行主機上，並配置好 SSH/HTTPS 憑證)
* **作業系統相容性**：Windows (PowerShell) / macOS / Linux

---

## 2. 系統架構與模組拆解 (System Architecture & Modules)

本系統採輕量化的**腳本觸發架構**，無複雜的伺服器與資料庫。主要模組如下：

### 2.1 Git 自動化控制器 (Git Automation Controller)
* **功能**：負責與本地 Git 客戶端互動。
* **命令集**：
  * 同步：`git pull`
  * 暫存：`git add .`
  * 提交：`git commit -m "Auto-update index"`
  * 推送：`git push`
* **異常捕獲**：檢查命令的 Exit Code，非 0 時拋出異常並阻斷流程。

### 2.2 目錄生成器 (Index Generator - `generate_index.py`)
* **功能**：掃描工作目錄，提取 HTML 檔案中的 Metadata（如 `<title>` 或 `<h1>`），並動態生成一個美觀的 `index.html` 首頁。
* **主要邏輯**：
  1. 遍歷指定目錄（排除 `.git`、`docs`、`.agents`、`.venv` 等系統或設計資料夾）。
  2. 讀取每個 HTML 檔案，利用 HTML 解析器（如 `BeautifulSoup` 或 Python 內建的 `html.parser`）抓取標題。
  3. 套用統一的 CSS 範本，渲染出目錄清單。
  4. 寫入到根目錄下的 `index.html`。

## 3. 目錄結構設計 (Directory Structure)

```text
github_html/
├── .agents/                 # AI Agent 專案規則配置
│   └── AGENTS.md
├── .git/                    # Git 版本庫
├── .venv/                   # Python 虛擬環境 (由 uv 自動管理)
├── docs/                    # 專案文件目錄
│   ├── sdd/
│   │   └── 系統設計文件.md
│   └── spec/
│       ├── 企劃書.md
│       └── 規格書.md
├── generate_index.py        # 目錄生成 Python 腳本
├── index.html               # 自動生成的網站目錄首頁 (產出物)
├── ProjectBrief .md         # 原始專案需求說明
└── task.md                  # 開發任務清單
```

---

## 4. 資料儲存與持久化 (Data Storage)
* **無傳統資料庫**。
* 所有的目錄資料均來自對檔案系統的**動態掃描**，最終的持久化產出即為 `index.html` 靜態檔案。
* 狀態管理完全依賴 Git 的 commit 歷史紀錄。
