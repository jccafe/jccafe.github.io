# github_html 系統設計文件 (SDD)

## 1. 技術選型 (Technology Stack)

* **運行環境**：Python 3.10+
* **環境與套件管理**：`uv` (極速 Python 虛擬環境與依賴管理器)
* **版本控制與自動化部署**：Git CLI (需要預先配置好 SSH/HTTPS 憑證)
* **雲端 CI/CD 工作流**：GitHub Actions (配置於 `.github/workflows/update-index.yml`)
* **控制台相容性**：Windows Console 自適應調整 (強制 stdout 以 UTF-8 輸出以防在 CP950 PowerShell 下發生 `UnicodeEncodeError`)
* **前端設計**：CSS 兩欄式 Sticky 側邊欄佈局、Glassmorphism (磨砂玻璃) 設計風格、RWD (自適應小螢幕 Tab 橫滑)、JavaScript Scrollspy 與平滑滾動。

---

## 2. 系統架構與模組拆解 (System Architecture & Modules)

本系統採**雙軌自動化架構**（本地一鍵部署 + 雲端事件觸發），以輕量化腳本為核心：

### 2.1 Git 自動化控制器 (Git Automation Controller - `deploy.py`)
* **功能**：負責在本地一鍵執行完整的 Git 同步與推送流程。
* **流程控制**：預檢工作區 -> 拉取遠端 -> 執行生成器 -> 檢查變更 -> 提交與推送。
* **異常攔截**：攔截非 Git 儲存庫、Pull 失敗以及 Git 合併衝突並提供繁體中文友善指引。

### 2.2 二階層式目錄生成器 (Index Generator - `generate_index.py`)
* **功能**：遍歷工作區下的 `.html` 檔案（排除排除目錄如 `.venv`、`docs` 等），以內建 `html.parser` 提取 `<title>`。
* **分群歸類**：依據網頁所在的實際資料夾路徑分群，以二階層式 Section 卡片呈現。
* **星期漸層主題**：內嵌 JS 偵測當前本地時間，自動切換對應星期之漸層背景。

### 2.3 雲端自動更新流 (GitHub Actions Workflow)
* **功能**：於 `.github/workflows/update-index.yml` 中定義，當有新網頁推送至儲存庫時，由雲端伺服器自動執行 `generate_index.py` 並將新 `index.html` 提交推送，徹底免手動本機執行。

│   └── AGENTS.md

│   ├── sdd/
│   │   └── 系統設計文件.md

│       ├── 企劃書.md

## 4. 資料儲存與持久化 (Data Storage)
* 系統完全無資料庫設計，屬**無狀態設計 (Stateless)**。
* 每次執行 `generate_index.py` 時都會進行一次完整的檔案系統目錄掃描。
* 資料的持久化載體為 `index.html`，版本狀態完全依賴 Git 的 commit 歷史紀錄。
