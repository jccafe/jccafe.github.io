# github_html 專案企劃書

## 1. 專案背景與目標
`jccafe.github.io` 作為一個個人或組織的靜態網站，隨著網頁與目錄數量的增加，手動更新網頁目錄（Index）變得繁瑣且容易出錯。
本專案 `github_html` 旨在開發一個**自動化網站目錄生成與部署助手**，透過一鍵（或自動化腳本）更新目錄，並自動將變更推送至 GitHub 儲存庫，確保網站內容的即時性與一致性。

## 2. 目標客群 (Target Audience)
* 靜態網站管理員（特別是使用 GitHub Pages 託管網站者）。
* 需要頻繁更新靜態目錄，並希望透過腳本自動化完成 Git 工作流的開發人員。

## 3. 核心價值 (Core Value)
* **省時高效**：將原本需要手動執行的「拉取代碼、執行目錄生成腳本、提交並推送代碼」等繁瑣步驟簡化為單一自動化指令。
* **降低出錯率**：避免因手動操作遺漏或指令順序錯誤導致的部署失敗或版本衝突。
* **流程標準化**：透過固定的工作流與例外處理機制（如衝突提醒），建立穩健的靜態網站更新流程。

## 4. 功能亮點 (Key Features)
* **自動同步機制**：執行前自動拉取最新版本（`git pull`），確保工作區乾淨且無衝突。
* **靜態目錄生成**：整合 `generate_index.py` 腳本，自動掃描並重新產生結構化的網站 HTML 目錄。
* **無縫 Git 部署**：自動追蹤變更並以標準格式（"Auto-update index"）進行 Commit 與 Push。
* **衝突防護機制**：當偵測到 Git 合併衝突時，能即時暫停並通知管理員手動介入，避免程式碼覆蓋。

## 5. 市場定位 (Positioning)
此專案定位為一個**輕量級的 Local-to-Remote 自動化 CI/CD 工具**，專為不需要複雜 Jenkins 或 GitHub Actions 流程的小型靜態網站提供最直覺、快速的目錄更新方案。
